## 一、项目概述

### 1.1 项目背景
在软件开发过程中，变量名、函数名、类名等通常由多个英文单词组合而成（如 `retryPolicy`、`build_config`）。对于英语非母语的开发者，理解这些命名需要频繁查询单词含义，且容易遗忘；编写代码时也常因词汇量不足而命名困难，影响代码可读性和开发效率。现有背单词软件缺乏编程语境，无法满足开发者的真实需求。

### 1.3 核心价值
- **语境学习**：在真实的代码上下文中收集单词，复习时关联原始场景，记忆更深刻。

**场景三：写代码时想用某个词，中文一搜即得**
当你需要给一个路由文件命名，想表达“主页路由”却一时想不起 `indexRouter` 时，无需切换到浏览器查词典。在 VS Code 中打开搜索框，输入“主页 路由”，你的个人词库立即返回 `indexRouter`，回车插入，行云流水。

团队版中，技术负责人维护一份“团队术语表”。新人加入时自动同步，代码审查时自动检查命名是否符合规范，让 `getUserData` 成为团队共识，而不是 `fetch_user_data` 和 `obtainUserData` 混用。
